# Scheme Generator (Генератор блок-схем Draw.io)

Генератор блок-схем — это CLI-утилита, предназначенная для автоматического построения визуальных схем в формате XML Draw.io на основе строгого текстового описания (YAML). Утилита автоматически выравнивает блоки, предотвращает перекрытия (наложения фигур) и прокладывает ортогональные связи, интеллектуально огибая хитбоксы элементов («умная маршрутизация»).

---

## 🤖 Работа через ИИ-агентов (Рекомендуемый способ)

Если вы хотите использовать LLM (ChatGPT, Claude, Copilot и др.) для автоматического перевода вашего кода в `.yaml` формат блок-схемы, **обязательно скормите агенту файл `prompt.md`**, лежащий в корне проекта. 

Этот файл содержит **системные инструкции** (пошаговое руководство, как разбивать код на слои и узлы) и **эталонные примеры генерации**, что позволит ИИ писать безупречные конфиги, которые генератор распознает и отрисовывает без ошибок с первого раза!

## 🚀 1. Использование (CLI)

Утилита вызывается из командной строки. Рекомендуется использовать менеджер зависимостей `uv` (установлены пакеты `PyYAML`, `pydantic`, `lxml`).

### Базовый синтаксис:
```bash
python -m scheme_generator.cli <input_file.yaml> [-o <output_file.drawio>]
```

### Аргументы:
*   `input_file.yaml` _(обязательный)_ — Путь к исходному конфигурационному файлу в формате YAML.
*   `-o` или `--output` _(необязательный)_ — Путь для сохранения результата. По умолчанию сохраняется как `output.drawio` в текущей директории.

## 2. Контракт YAML формата

Для описания схемы используется строгий формат YAML. Данные разбиты на три обязательных корневых ключа: `meta`, `nodes` и `edges`.

### 2.1 Блок `meta`
Содержит общие метаданные схемы.
*   `name` (String, обязательно): Имя схемы (используется как название страницы внутри файла Draw.io).
*   `description` (String, опционально): Описание алгоритма для документирования файла.

### 2.2 Блок `nodes` (Узлы)
Словарь всех узлов схемы. Ключ — уникальный строковый идентификатор узла (ID). Разрешены только латинские буквы, цифры и `_`.

| Атрибут | Тип | Обязательный | Описание |
| :--- | :--- | :---: | :--- |
| `type` | Integer | Да | Числовой идентификатор типа фигуры (от 1 до 6). См. Маппинг типов. |
| `layer` | Integer | Да | Индекс горизонтального слоя (сверху вниз). `0` — самый верхний слой. Код, который выполняется раньше, находится в слое с меньшим индексом. Все узлы с одинаковым `layer` отображаются на одной высоте. |
| `text` | String | Да | Текст внутри блока. Поддерживает многострочность через оператор `\|`. **Индексы массивов** задаются синтаксисом `_{i}`, что транслируется в подстрочный HTML тег (например `arr_{i}` -> `arr`<sub>`i`</sub>). |
| `order` | Integer | Нет | Порядок отрисовки блока слева направо внутри одного слоя. Если не указать, блоки сортируются автоматически. |

**Маппинг типов блоков (`type`):**
1.  **Действие (1)** — Прямоугольник. Используется для простых линейных вычислений и присвоений.
2.  **Условие (2)** — Горизонтальный ромб. Проверяемое условие (if). Требует подписи "Да"/"Нет" на исходящих ребрах.
3.  **Вызов функции (3)** — Прямоугольник с засечками (process). Вызовы подпрограмм или методов.
4.  **Данные ввод/вывод (4)** — Параллелограмм. Чтение входных аргументов, возврат (return) или выброс `Exception`.
5.  **Начало / Конец (5)** — Эллипс/Овал. Стартовые и финишные терминальные блоки.
6.  **Цикл (6)** — Шестиугольник. Блок инициализации цикла (вид: `i = 0, n - 1`). 

### 2.3 Блок `edges` (Связи)
Список ориентированных ребер. Определяет переходы между блоками (строго направленные линии).

| Атрибут | Тип | Обязательный | Описание |
| :--- | :--- | :---: | :--- |
| `from` | String | Да | ID исходного узла (совпадает с ключом в блоке `nodes`). |
| `to` | String | Да | ID целевого узла. |
| `label` | String | Нет | Текстовая подпись ребра (например, "Да", "Нет"). |

## 3. Умная маршрутизация связей (Smart Layout Engine v3)

Программа автоматически рассчитывает размеры блоков на основании длины текстов и вычисляет координаты для них так, чтобы они не накладывались друг на друга.

Механизм построения соединительных линий работает по принципам интеллектуального избегания коллизий (хитбоксов):
1.  **Линейный код (Layer -> Layer+1):** Маршрутизируется строго вертикально, блок под блоком.
2.  **Прыжки/Исключения (Layer -> Layer+5):** Если ребро проскакивает через несколько слоёв (ранний выход или прерывание), алгоритм вычисляет Bounding Boxes всех промежуточных блоков на пройденных слоях. Он находит оптимальный вертикальный "безопасный коридор" (свободную `X` координату) и выставляет туда прозрачные контрольные точки. Линия получается идеально прямой и не проходит сквозь другие блоки.
3.  **Обратные связи / Циклы:** При возврате линии на верхний слой (к началу цикла), ребро автоматически переводится на внешнее обходное кольцо с правой стороны схемы.

## 4. Пример конфигурации (test_1.yaml)

Этот комплексный YAML файл полностью описывает логику нормализации оценок с циклами, ветвлениями и множественными точками выхода (exception и success). Вышеназванная SVG картинка была сгенерирована на его основе:

```yaml
meta:
  name: "ProcessStudentGrades"
  description: "Комплексный алгоритм нормализации оценок с множественным выходом"

nodes:
  # --- СЛОЙ 0 ---
  start_node:
    type: 5
    layer: 0
    text: "НАЧАЛО"

  # --- СЛОЙ 1 ---
  input_data:
    type: 4
    layer: 1
    text: "grades, n"

  # --- СЛОЙ 2 ---
  check_empty:
    type: 2
    layer: 2
    text: "n <= 0"

  # --- СЛОЙ 3 ---
  init_vars:
    type: 1
    layer: 3
    order: 1
    text: |
      passed_count = 0
      i = 0
      
  err_empty:
    type: 4
    layer: 3
    order: 2
    text: "Exception: No students data"

  # --- СЛОЙ 4 ---
  loop_node:
    type: 6
    layer: 4
    text: "i = 0, n - 1"

  # --- СЛОЙ 5 ---
  func_call:
    type: 3
    layer: 5
    text: "normalized = apply_curve(grades_{i})"

  # --- СЛОЙ 6 ---
  check_pass:
    type: 2
    layer: 6
    text: "normalized >= 50"

  # --- СЛОЙ 7 ---
  update_stats:
    type: 1
    layer: 7
    text: |
      passed_count = passed_count + 1
      grades_{i} = normalized

  # --- СЛОЙ 8 ---
  check_any_passed:
    type: 2
    layer: 8
    text: "passed_count > 0"

  # --- СЛОЙ 9 ---
  out_success:
    type: 4
    layer: 9
    order: 1
    text: "grades, passed_count"

  err_all_failed:
    type: 4
    layer: 9
    order: 2
    text: "Exception: All students failed"

  # --- СЛОЙ 10 ---
  end_node:
    type: 5
    layer: 10
    text: "КОНЕЦ"


edges:
  - from: start_node
    to: input_data
  - from: input_data
    to: check_empty
  - from: check_empty
    to: err_empty
    label: "Да"
  - from: check_empty
    to: init_vars
    label: "Нет"
  - from: init_vars
    to: loop_node
  - from: loop_node
    to: func_call
  - from: func_call
    to: check_pass
  - from: check_pass
    to: update_stats
    label: "Да"
  - from: update_stats
    to: loop_node
  - from: check_pass
    to: loop_node
    label: "Нет"
  - from: loop_node
    to: check_any_passed
  - from: check_any_passed
    to: out_success
    label: "Да"
  - from: check_any_passed
    to: err_all_failed
    label: "Нет"
  - from: err_empty
    to: end_node
  - from: out_success
    to: end_node
  - from: err_all_failed
    to: end_node
```
