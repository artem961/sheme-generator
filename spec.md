# Спецификация генератора блок-схем

Данный документ описывает требования к визуальному представлению блок-схем и строгий формат конфигурационного файла для их автоматической генерации в XML-формат Draw.io.

---

## СЕКЦИЯ 1. Общие требования к блок-схемам

Архитектура схемы базируется на графовой модели с привязкой элементов к горизонтальным слоям (Layers) для отображения линейного исполнения кода. 

### 1.1 Правила построения и слои
*   **Линейность и слои:** Код, который выполняется раньше, находится в слое с меньшим индексом (Слой 0 — самый верхний). Элементы одного слоя располагаются строго на одном горизонтальном уровне.
*   **Связи между слоями:** Блоки могут соединяться между любыми слоями. Допускаются переходы через несколько слоев вниз (пропуск кода), а также переходы на слои вверх (для циклов).
*   **Множественные связи:** Блоки могут иметь более одного предка (входящие соединения) и более одного наследника (исходящие соединения).
*   **Предотвращение наложений:** Блоки, находящиеся на одном слое, должны автоматически распределяться по горизонтали (оси X), чтобы не перекрывать друг друга и линии связей.
*   **Форматирование текста:** Текст схемы должен содержать только строгие логические действия и код. **Никаких комментариев или пояснений.**
*   **Индексы массивов:** Индексы должны отображаться в формате нижнего регистра (подстрочный текст, например: $a_i$), использование квадратных скобок (типа `a[i]`) строго запрещено.

### 1.2 Ребра (Соединения)
*   Это направленные линии (стрелки), строго соединяющие начало одного блока с концом другого. 
*   Могут содержать текстовую подпись (в основном для обозначения истинной/ложной ветви условий).

### 1.3 Типы блоков и их требования

1.  **Действие (Прямоугольник).**
    *   Описывает простые линейные операции.
    *   Каждое новое действие должно записываться с новой строки внутри одного блока.
2.  **Условие (Горизонтальный ромб).**
    *   Содержит проверяемое логическое условие.
    *   Должен иметь исходящие ребра с подписями («Да» / «Нет» или «True» / «False»).
    *   Допускается наличие только одного исходящего ребра, если в коде отсутствует ветвь `else`.
3.  **Вызов функции (Прямоугольник с засечками по бокам).**
    *   Содержит название вызываемой функции, переданные аргументы.
    *   При необходимости содержит присвоение результата работы функции целевой переменной.
4.  **Входящие/выходящие данные (Параллелепипед).**
    *   **Входные данные:** Должен стоять строго после блока начала программы и быть соединенным с ним.
    *   **Выходные данные:** Должен стоять строго перед блоком конца программы. Может содержать возвращаемые данные или `Exception` (завершение с ошибкой). 
    *   Может быть несколько блоков выходных данных (от разных ветвлений), но все они обязаны сводиться (соединяться) к финальному блоку «Конец».
5.  **Начало / Конец программы (Овал).**
    *   Специальный блок. Каждая отдельная блок-схема (метод) обязана иметь строго один блок начала.
    *   Каждая блок-схема обязана завершаться блоком (или блоками, если их несколько) конца. 
    *   Блок начала обозначает передачу данных в метод.
6.  **Блок цикла (Вытянутый шестиугольник).**
    *   Текст блока содержит переменную цикла и перебираемый диапазон строго в формате: `i = 0, n - 1`.
    *   **Вход в тело:** Имеет исходящее ребро, уходящее к слою с более высоким номером (внутрь тела цикла).
    *   **Возврат:** Последний блок тела цикла обязан иметь исходящее соединение, возвращающееся обратно в данный блок цикла.
    *   **Выход из цикла:** При завершении перебора, блок цикла должен иметь ребро, указывающее на последующий код (на слой ниже тела цикла).

## СЕКЦИЯ 2. Спецификация формата описания (YAML)

Для описания схемы используется строгий формат YAML. Данные разбиты на три корневых ключа: `meta`, `nodes` и `edges`.

### 2.1 Корневая структура

```yaml
meta:          # Метаданные схемы (информационно)
  name: "string"
nodes:         # Словарь всех узлов схемы
  [node_id]: { ...NodeObject }
edges:         # Список связей между узлами
  - { ...EdgeObject }
```

### 2.2 Спецификация объекта Узла (NodeObject)

Каждый узел объявляется как ключ в словаре `nodes`. Ключ (id) узла должен состоять только из латинских букв, цифр и нижних подчеркиваний.

| Параметр | Тип | Обязательный | Описание |
| :--- | :--- | :---: | :--- |
| `type` | Integer | Да | Тип блока от 1 до 6 (см. маппинг типов ниже). |
| `layer` | Integer | Да | Номер слоя по оси Y ($\ge 0$). 0 — самый верх. |
| `text` | String | Да | Содержимое блока. Многострочный текст поддерживается через оператор `\|`. |
| `order` | Integer | Нет | Порядок отрисовки слева-направо внутри одного слоя (от меньшего к большему). Если не указан, транслятор распределяет их автоматически. |

**Маппинг типов (type):**
*   `1` — Действие
*   `2` — Условие
*   `3` — Вызов функции
*   `4` — Данные (ввод/вывод)
*   `5` — Начало/Конец
*   `6` — Цикл

**Спецификация форматирования текста:**
*   Для многострочных блоков (особенно тип 1) используется YAML-синтаксис `|`:
    ```yaml
    text: |
      x = a + b
      y = c + d
    ```
*   **Индексы массивов** описываются строго через нижнее подчеркивание и фигурные скобки: `_{индекс}`.
    *   *Пример:* `a_{i} = b_{i+1}` транслятор превратит в подстрочный текст: a<sub>i</sub> = b<sub>i+1</sub>.

### 2.3 Спецификация объекта Связи (EdgeObject)

Элемент списка `edges`. Описывает строгую направленную связь от одного блока к другому.

| Параметр | Тип | Обязательный | Описание |
| :--- | :--- | :---: | :--- |
| `from` | String | Да | ID узла-источника. |
| `to` | String | Да | ID целевого узла. |
| `label` | String | Нет | Текстовая подпись ребра (например, "Да" / "Нет"). |

### 2.4 Пример правильной записи полного алгоритма

```yaml
meta:
  name: "FindMaxInArray"
  
nodes:
  # --- СЛОЙ 0 ---
  start_node:
    type: 5
    layer: 0
    text: "НАЧАЛО"

  # --- СЛОЙ 1 ---
  input_data:
    type: 4
    layer: 1
    text: "arr, n"

  # --- СЛОЙ 2 ---
  init_vars:
    type: 1
    layer: 2
    text: |
      max_val = arr_{0}
      i = 1

  # --- СЛОЙ 3 ---
  loop_start:
    type: 6
    layer: 3
    text: "i = 1, n - 1"

  # --- СЛОЙ 4 ---
  check_max:
    type: 2
    layer: 4
    text: "arr_{i} > max_val"

  # --- СЛОЙ 5 ---
  update_max:
    type: 1
    layer: 5
    text: "max_val = arr_{i}"

  # --- СЛОЙ 6 ---
  print_func:
    type: 3
    layer: 6
    order: 1 # Выстраивание узлов на одном слое
    text: "print(max_val)"
    
  error_func:
    type: 4
    layer: 6
    order: 2
    text: "Exception: array is empty"

  # --- СЛОЙ 7 ---
  end_node:
    type: 5
    layer: 7
    text: "КОНЕЦ"


edges:
  # Линейный старт
  - from: start_node
    to: input_data
  - from: input_data
    to: init_vars
  - from: init_vars
    to: loop_start

  # Вход в тело цикла
  - from: loop_start
    to: check_max

  # Ветвление внутри цикла
  - from: check_max
    to: update_max
    label: "Да"

  # Возврат в начало цикла из ветви "Да"
  - from: update_max
    to: loop_start
  
  # Возврат в начало цикла из ветви "Нет" (пропуск update_max)
  - from: check_max
    to: loop_start
    label: "Нет"

  # Выход из цикла (конец перебора)
  - from: loop_start
    to: print_func

  # Соединение выходов с концом
  - from: print_func
    to: end_node
  - from: error_func
    to: end_node
```

### 2.5 Инструкции для будущего транслятора (Draw.io XML)

1.  **Координата Y:** Вычисляется строго на основе свойства `layer`. Базовая формула: `Y = layer * LAYER_HEIGHT_STEP`.
2.  **Координата X:** Парсер группирует все ID узлов с одинаковым значением `layer`. Если узлов несколько, они распределяются относительно центра по оси X. Свойство `order` определяет сортировку слева-направо. Если `order` нет — сортировка в алфавитном порядке ID.
3.  **Обработка текста:** Парсер обязан пропускать поле `text` через Regex-замену: заменять паттерн `_\{([^}]+)\}` на XML-теги `<sub>$1</sub>` (или HTML эквивалент, поддерживаемый Draw.io).
4.  **Маршрутизация линий (Edges):** Для обратных связей (от большего `layer` к меньшему) парсер должен задавать тип пути Draw.io (например, ортогональная маршрутизация "Orthogonal Edge Style"), чтобы линия аккуратно огибала блоки по бокам, не пересекая их текст.
